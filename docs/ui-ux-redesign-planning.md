# VKR — карта страниц и UX-аудит (планирование редизайна)

> Документ-основа для переработки UI/UX. Часть 1 — что есть на каждой странице
> сейчас. Часть 2 — аудит проблем и рекомендации. Стек: Nuxt 4 + Vue 3 +
> `@nuxt/ui` v4 + Tailwind v4, интерфейс на русском.

---

## Часть 1. Карта страниц

## 1. Публичная зона (лендинг + вход)

### `/` — Лендинг (`pages/index.vue`, layout `landing`)

Маркетинговая главная. Шапка (`UHeader`): логотип «Классный журнал», меню
(Возможности / Как это работает / Вопросы), выбор языка, переключатель темы,
кнопка «Войти». Секции:

- **Hero** — заголовок «Самостоятельная отметка и оценивание студентов»,
  описание, кнопки «Войти» и «Как это работает». Справа — демо-карточка с
  QR-кодом, бейджем «Активна» и списком студентов (бейджи «Вовремя/Опоздал»).
- **Возможности** — сетка из 8 карточек с иконками.
- **Как это работает** — 3 карточки-шага.
- **Деталь: отметка по QR** — карточка-мокап с QR и временем отметок.
- **Деталь: таблица посещаемости** — мокап матрицы 5×3 + легенда.
- **FAQ** — аккордеон из 5 вопросов.
- **CTA** — финальный призыв с кнопкой «Войти».
- **Footer** — копирайт.

### `/auth/login` — Вход (`pages/auth/login.vue`)

Центрированный экран: логотип, карточка «С возвращением», большая кнопка «Войти»
(OIDC SSO), подсказка, ссылка «На главную». Справа сверху — язык и тема.

### `/check-in/[id]` — Публичная отметка студента (`pages/check-in/[id].vue`, layout `landing`)

Открывается студентом по QR/ссылке без авторизации. Поведение по состоянию сессии:

- Заголовок с темой занятия + бейдж состояния.
- При активной сессии — карточка с двумя таймерами (основное окно / окно для опоздавших).
- **Шаг 1:** ввод 6-значного кода аудитории (`UPinInput`).
- **Шаг 2:** поиск себя по фамилии → карточки студентов с кнопкой «Это я» → модалка подтверждения.
- После отметки — зелёный алерт (блокировка повторной отметки через localStorage).
- Закрытая/отменённая сессия — нейтральная заглушка.

## 2. Рабочий стол (dashboard)

**Каркас** (`layouts/dashboard.vue`): слева сворачиваемый/растягиваемый сайдбар
(переключатель предмета вверху, навигация: Главная, Предметы с раскрытием списка,
Группы студентов, Feedback в Telegram, внизу меню пользователя). Справа — панель с
навбаром (хлебные крошки), тулбаром и телом.

### `/dashboard` — Главная (`dashboard/index.vue`)

Сетка карточек-ссылок на разделы (дашборд-лаунчер).

### `/dashboard/subjects` — Список предметов (`subjects/index.vue`)

Заголовок + Обновить/Создать. Поиск. Сетка карточек предметов. Пагинация.
Состояния: скелетоны / пусто / ошибка.

### `/dashboard/subjects/create` — Создание предмета

Форма (название/описание).

### `/dashboard/subjects/[uuid]` — Обзор предмета (`subjects/[uuid]/index.vue`)

Заголовок «Обзор предмета». Сетка карточек: Занятия, Отметка присутствия,
Посещаемость, Оценки, Итоговые оценки, и по правам — Настройки. Без назначения —
пустое состояние «Назначения нет».

## 3. Внутри предмета

### `/.../lessons` — Занятия (`lessons/index.vue`)

Заголовок + Обновить / Excel / «По расписанию» / «По количеству». Инфо-алерты

- `lessons/create` — генерация по количеству (лекции + практики).
- `lessons/schedule-create` — по недельному расписанию.
- `lessons/[id]/edit` — редактирование занятия.
- `lessons/[id]/scopes-create` — проведения (группы/подгруппы + даты).
- `lessons/[id]/assignments-create` / `assignments-edit` — задания.

### `/.../lessons/[id]` — Карточка занятия (`lessons/[id]/index.vue`)

Заголовок с темой, бейдж «Активное», Назад/Обновить/Редактировать. Три инфо-карточки
(Тип, Аудитория и даты, Задания). Табы Посещаемость / Оценки с инлайн-редактируемыми
таблицами, Сбросить/Сохранить, защита от ухода с несохранёнными изменениями.

### `/.../check-ins` — Сессии отметки (`check-ins/index.vue`)

Заголовок + «Запустить опрос». Инфо-алерт. Фильтры по группе/подгруппе. Таблица
сессий с rowspan по сессии: состояние, время, окна, подтверждение, группа,
подгруппа. Пустое состояние с CTA.

### `/.../check-ins/create` — Запуск опроса

Форма (выбор занятия, окна «вовремя»/«опоздавших»).

### `/.../check-ins/[id]` — Живая сессия преподавателя (`check-ins/[id].vue`)

Самая насыщенная страница. Карточка состояния (бейдж, время, аудитория, окна с
обратным отсчётом, «Отменить»). Карточка «Отправить студентам» (большой QR, крупный
код, ссылка с копированием/шарингом/скачиванием). Карточка «Текущее состояние»
(живые счётчики + таблица в реальном времени, поллинг 5 с). Карточка «Подтверждение
результатов» (таблица предлагаемых статусов + комментарии → «Подтвердить и
перенести»). Финальные алерты «Подтверждена»/«Отменена».

### `/.../attendances` — Посещаемость (`attendances/index.vue`)

Заголовок + экспорт Excel (дропдаун). Инфо-алерт. Фильтры: табы Все/Лекции/Практики,
мультиселект групп, сортировка, переключатель «Редактирование» + Сбросить/Сохранить.
Секционированная матрица студенты × занятия с цветными статусами, инлайн-редактирование,
подсветка по политике. Защита от ухода.

### `/.../grades` — Оценки (`grades/index.vue`)

Близнец посещаемости: те же фильтры/экспорт/режим редактирования. Таблица — журнал
баллов по занятиям/заданиям с учётом штрафов/бонусов (исходный балл в тултипе),
пометки обязательных заданий, настраиваемая подсветка.

### `/.../final` — Итоговые оценки (`final/index.vue`)

Заголовок + экспорт. Инфо-алерт. Фильтры. Две легенды (Вердикт, Банды). По секциям —
заголовок со статистикой (средний/макс/мин), «на весь экран», таблица итоговых сумм с
колонкой «Вердикт». Полноэкранный режим в `UModal`.

### `/.../settings` — Настройки предмета (`settings/index.vue`)

Сгруппированные карточки: Доступ (назначения), Оценивание (штрафы/бонусы, аттестация),
Посещаемость (учёт, единое время), Отображение (подсветка таблиц).

- `permissions/` + `create` + `[id]/edit` — назначения преподавателей.
- `penalty-policy/` — штрафы и бонусы.
- `final-assessment-policy/` — уровни аттестации и порог посещаемости.
- `attendance-policy/` — баллы за посещаемость.
- `check-in-policy/` — единые окна отметки.
- `grading-highlight-policy/` + `attendance-highlight-policy/` — раскраска таблиц.

## Часть 2. UX-аудит

Приоритеты: 🔴 высокий (бьёт по основным сценариям), 🟡 средний, 🟢 низкий
(полировка). Для каждой проблемы — рекомендация.

## A. Сквозные проблемы (затрагивают много страниц)

### 🔴 A1. Инфо-алерты «Зачем это нужно» на каждой странице

Почти каждая страница начинается с серого `UAlert` с длинным пояснительным текстом.
Для нового пользователя — польза, для ежедневной работы — постоянный визуальный шум,
отодвигающий рабочий контент (таблицы, кнопки) вниз.

- **Рекомендация:** сделать пояснения сворачиваемыми/закрываемыми с запоминанием в
  localStorage; либо вынести в иконку-«?» рядом с заголовком (поповер), либо в
  онбординг-тур. Контент по умолчанию — выше «над сгибом».

### 🔴 A2. Дублирование экранов «Посещаемость» и «Оценки»

Две страницы почти идентичны по логике (фильтры, экспорт, режим редактирования,
секционированная таблица), но это два отдельных файла с копипастой. Для пользователя
это означает переключение контекста между почти одинаковыми экранами.

- **Рекомендация (UX):** объединить в один экран «Журнал» с переключателем
  Посещаемость / Оценки (как уже сделано на странице занятия). Меньше пунктов в
  меню, единый ментальный паттерн.
- **Рекомендация (код):** общий компонент-обёртка `JournalTable` с фильтрами и
  тулбаром; страницы становятся тонкими.

### 🟡 A3. Кнопка «Обновить» вручную везде

На каждой странице есть `i-lucide-refresh-cw`. Это сигнал, что пользователь не
доверяет актуальности данных. На живых экранах (check-in) уже есть поллинг.

- **Рекомендация:** авто-refresh при фокусе вкладки/возврате на страницу
  (`useIntervalFn` + `visibilitychange`), оставить ручную кнопку как запасную и
  показывать «обновлено N мин назад».

### 🟡 A4. Непоследовательность вариантов и расположения кнопок

Экспорт Excel где-то `variant="outline"`, где-то `variant="ghost"`, где-то дропдаун,
где-то одиночная кнопка. «Обновить» то ghost-иконка, то с лейблом.

- **Рекомендация:** дизайн-токены для типовых действий (primary action / secondary /
  destructive / refresh / export) и единое правило порядка в тулбаре
  (слева — контекст, справа — действия; деструктивные — отдельно).

### 🟡 A5. Терминология «Банды», «Скоупы», «Вердикт»

В итоговой таблице легенда «Банды» — калька с `bands`, непонятная преподавателю.
«Проведение», «scope» — внутренний жаргон просочился в UI.

- **Рекомендация:** ревизия всех пользовательских строк, замена на человеческие
  термины («Уровни оценки» вместо «Банды», «Проведение занятия» с пояснением).

### 🟢 A6. Локализация только частичная

Лендинг и логин — en+ru, остальное приложение — хардкод русского.

- **Рекомендация:** если многоязычность нужна — вынести строки дашборда в i18n;
  если нет — убрать `AppLocaleSelect` из публичной зоны, чтобы не обещать то, чего
  нет внутри.

### 🟢 A7. Дев-артефакты в проде

Карточка «Test» и страница `subjects/[uuid]/test.vue` (`import.meta.dev`).

- **Рекомендация:** убедиться, что не попадает в прод-бандл; вынести из `pages/`.

## B. Навигация и информационная архитектура

### 🔴 B1. Глубокая вложенность настроек

Путь до политики: Предмет → Настройки → конкретная политика — 3+ клика, у каждой
политики своя страница-форма. Между политиками нельзя быстро переключаться.

- **Рекомендация:** на странице настроек — боковые вкладки (master-detail): слева
  список разделов, справа форма. Не теряем контекст предмета, переключение в 1 клик.

### 🟡 B2. Сайдбар: список предметов внутри пункта меню

Предметы раскрываются вложенным списком (max-height 56, скролл). При большом числе
предметов — неудобный мелкий скролл внутри меню.

- **Рекомендация:** командная палитра (Ctrl+K) для быстрого перехода к предмету/группе;
  либо переключатель предмета вверху сайдбара уже есть — сделать его основным, а из
  меню убрать дубль.

### ✅ B3. Хлебные крошки — уже реализованы

Оказалось, что сквозные крошки уже есть: родительские route-файлы
`pages/dashboard/subjects.vue` и `groups.vue` собирают `UBreadcrumb` для всех
дочерних маршрутов + секционное меню предмета в тулбаре. Пункт закрыт.

- **Мелкий остаток:** нет листовой крошки для `lessons/schedule-create` и карточки
  занятия `lessons/[id]` — можно дочистить.

### 🟢 B4. «Feedback» в Telegram в основном меню

Ссылка на личный Telegram (`k1mbb.t.me`) в постоянной навигации.

- **Рекомендация:** перенести в меню пользователя/футер; не держать в первичной навигации.

## C. Таблицы (ядро продукта)

### 🟡 C1. Плотность и читаемость больших матриц

Посещаемость/оценки/итоги — широкие матрицы студенты × занятия.

- **Уже есть:** первый столбец (имя) и шапка закреплены (`sticky left-0` + `sticky`
  header во всех трёх таблицах), полноэкранный режим секции.
- **Остаётся:** hover-подсветка всей строки и столбца (cross-hair); компакт/комфорт
  режим плотности; заметная индикация горизонтального скролла на узких экранах.

### 🔴 C2. Инлайн-редактирование: дискаверабилити и масштаб

Чтобы редактировать, нужно найти переключатель «Редактирование». Массовых операций
(проставить всем «присутствовал», заполнить столбец) нет — только по ячейке.

- **Рекомендация:** действия над столбцом/строкой (выпадающее меню в шапке: «всем
  присутствовал», «очистить»), выделение диапазона, копирование значения вниз. Явный
  индикатор «режим редактирования включён» (рамка/баннер).

### 🟡 C3. Цветовая подсветка и доступность

Статусы и подсветка строятся на цвете (зелёный/жёлтый/красный). Для дальтоников и при
печати — теряется смысл.

- **Рекомендация:** дублировать цвет иконкой/буквой (П/О/Н), проверить контраст по
  WCAG AA, режим «без цвета» для печати/экспорта.

### 🟡 C4. «Группы не выбраны» как пустое состояние

Если снять все секции в фильтре — таблица исчезает с `UEmpty`. Легко случайно «потерять»
данные.

- **Рекомендация:** не давать снять последнюю секцию или показывать ненавязчивый
  баннер «фильтр скрывает все группы — сбросить» вместо полного пустого экрана.

## D. Ключевые сценарии

### ✅ D1. Сценарий «провести занятие» — добавлен чеклист готовности

Было: 5–6 экранов без ведущего флоу; карточка занятия лишь реактивно показывала
«Проведение не назначено» / «Задания не назначены».

- **Сделано:** компонент `LessonsReadinessChecklist` на карточке занятия — шаги
  «Проведение назначено» → «Задания добавлены» → «Отметка присутствия» с галочками,
  счётчиком «N из 3» и контекстными кнопками действий. Шаг отметки знает состояние
  (не начата / идёт / проведена) по сессиям занятия. Дублирующие кнопки из
  инфо-карточек убраны — действия теперь в одном месте.
- **Остаётся (по желанию):** довести до полноценного мастера с шагом «выставить
  оценки», прогресс по всем занятиям предмета.

### 🟡 D2. Запуск check-in оторван от занятия

`check-ins/create` — отдельный раздел, занятие выбирается внутри. Логичнее запускать
отметку из контекста конкретного занятия.

- **Рекомендация:** кнопка «Отметка присутствия» на карточке занятия с
  предзаполненным занятием; раздел «Сессии» оставить как историю/журнал.

### 🟢 D3. Подтверждение результатов: ручная вычитка длинного списка

На `check-ins/[id]` в состоянии «Ожидает подтверждения» — таблица со `USelect` в каждой
строке. Для большой группы — долго.

- **Рекомендация:** массовые действия («всех не отметившихся → отсутствует» уже
  дефолт, добавить «принять все предложенные»), сортировка «спорные сверху».

## E. Адаптивность и мелочи

### 🟡 E1. Широкие таблицы на мобильных/планшетах

Матрицы на узком экране — сильный горизонтальный скролл. Преподаватель в аудитории
часто с телефона/планшета.

- **Рекомендация:** на мобильном — карточный режим (один студент = карточка с
  занятиями) или приоритет колонок; продумать таргет тач-целей в ячейках.

### 🟢 E2. QR-карточка: фиксированный белый фон

В live-сессии и на лендинге QR на `bg-white dark:bg-white`. Осознанно (контраст
сканера), но визуально выбивается в тёмной теме.

- **Рекомендация:** оформить как намеренную «карточку для проектора» с рамкой/подписью.

### 🟢 E3. Пустые состояния с одинаковым заголовком

Напр., в списке предметов оба варианта `:title` — «Предметы не найдены» (тернарник без
разницы веток).

- **Рекомендация:** вычитать тексты пустых состояний, развести «ничего не найдено по
  фильтру» и «здесь пока пусто, создайте первый».

4. Командная палитра Ctrl+K для перехода к предмету/группе (B2).

## Реализовано в этой итерации (быстрые победы)

- **A1 — сворачиваемые подсказки.** Новый компонент `AppHint.vue`: инфо-алерт с
  крестиком, закрытие запоминается в localStorage (`hint-dismissed:<id>`). Применён
  на 5 ежедневных экранах: занятия, сессии отметки, посещаемость, оценки, итоги.
- **D2 — запуск отметки с занятия.** Форма запуска принимает `?lessonId=` и
  предвыбирает проведение, если для занятия доступно ровно одно. Точка входа теперь —
  чеклист готовности (см. D1), а не отдельная кнопка в шапке.
- **D1 — чеклист готовности занятия.** `LessonsReadinessChecklist`: шаги проведения /
  заданий / отметки с галочками, счётчиком и контекстными действиями; статус отметки
  вычисляется по сессиям занятия.
- **B2 — командная палитра (Ctrl/⌘+K).** `UDashboardSearch` + кнопка
  `UDashboardSearchButton` в сайдбаре; группы «Разделы» и «Предметы» из
  `useDashboardNavigation` (без лишних запросов — переиспользуются загруженные предметы).
- **A3 — авто-обновление при возврате на вкладку.** `useRefreshOnFocus`
  (visibilitychange/focus + троттлинг). На списках — без условий; на
  посещаемости/оценках — с гардом (не в режиме правки и без несохранённых изменений).
- **B1 — настройки master-detail.** Родительский `settings.vue`: слева навигация по
  политикам, справа форма; на обзоре навигация скрыта, показаны карточки.
- **B4 + чистка меню пользователя.** Из `UserMenu` убраны английские mock-пункты
  (Profile/Settings/Docs/Notifications…); меню переведено, оставлены рабочие пункты —
  тема, обратная связь (перенесена сюда из основной навигации), выход.
- **A5 — термины.** «Банды» → «Уровни» в легенде итогов.
- **E3 — пустые состояния.** Разведены заголовки «ничего не найдено» и «пока пусто»
  в списке предметов.
- **Уточнения аудита.** B3 (крошки), ядро C1 (sticky-столбец/шапка), C3 (иконка+буква
  в статусах посещаемости) и часть C2 (действие «Всем: …» в шапке столбца
  посещаемости) уже были в проекте — отмечено выше.

> Проверено: `pnpm typecheck`, `pnpm lint` и `pnpm build` проходят. Визуально в
> браузере не гонялось — стоит открыть `pnpm dev` и пройти основные экраны.

## Осталось (крупное — требует решения/визуальной проверки)

- **A2 — единый «Журнал» (посещаемость + оценки).** Самый частый экран, но полное
  слияние двух маршрутов рискованно без визуальной проверки; навигационную стоимость
  уже снижает секционное меню предмета (переключение в один клик). Делать отдельным
  заходом с прототипом.
- **E1 — карточный режим таблиц на мобильных.** Широкие матрицы на телефоне; нужен
  дизайн адаптива (карточка = студент).
- **C1 (остаток)** — hover cross-hair и режим плотности. **C2 (grades)** — массовые
  действия по столбцу в оценках (в посещаемости уже есть). **D3** — «принять все
  предложенные» в подтверждении отметки. **A4** — единые токены кнопок.
